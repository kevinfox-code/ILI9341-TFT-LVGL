# ILI9341-TFT-LVGL — STM32F446RE + LVGL v9 + FreeRTOS Reference Project

A working reference application for the 2.8" ILI9341 SPI TFT with XPT2046
resistive touch, built on an STM32F446RE (Nucleo-64), LVGL v9.2.x, and
FreeRTOS (CMSIS-RTOS2). The point of this repository is this project: a real,
buildable app that shows how to wire CubeMX-generated code, a hand-written
`App/` layer, and a CubeMX-decoupled `tft_drivers` driver library together
cleanly.

**Tested on:** STM32F446RE (Nucleo-64)
**LVGL version:** 9.2.x
**HAL:** STM32 HAL (CubeMX-generated), reached only through `App/drv_constants.h`

## Project layout

```text
ILI9341-TFT-LVGL/
├── CMakeLists.txt              # top-level glue — never regenerated by CubeMX
├── CMakePresets.json           # Debug / Release presets (Ninja + arm-none-eabi)
├── App/                        # hand-written application layer
│   ├── CMakeLists.txt          # drv_platform contract, LVGL, tft_drivers, app sources
│   ├── drv_constants.h         # THE bridge: DRV_*/APP_* macros → CubeMX symbols
│   ├── app_main.c / .h         # App_Init() / App_CreateTasks(), called from main.c
│   ├── system.c / .h           # RTC/UART glue — the only other file touching raw handles
│   ├── gui/                    # LVGL screens
│   ├── lv_conf.h, lvgl/        # vendored LVGL source + config
│   └── drivers/                # the tft_drivers library (see its own README)
│       ├── CMakeLists.txt      # tft_drivers static-library target (+ options)
│       ├── include/drv/, src/, adapters/, templates/, cmake/
│       ├── tests/              # host unit tests + hardware smoke checklist
│       └── datasheets/
└── Profiles/
    └── board_1/                # 100% CubeMX-generated output — never hand-edited
        ├── board_1.ioc
        ├── Core/ (Inc, Src)    # main.c, stm32f4xx_it.c, FreeRTOSConfig.h, ...
        ├── Drivers/ (CMSIS, STM32F4xx_HAL_Driver)
        ├── Middlewares/ (FreeRTOS)
        ├── cmake/stm32cubemx/CMakeLists.txt
        ├── startup_stm32f446xx.s
        └── STM32F446XX_FLASH.ld
```

**Rule:** nothing outside `Profiles/board_1/` is ever touched by CubeMX
regeneration, and the only lines added inside it live in `USER CODE` blocks —
`Core/Src/main.c` and `Core/Src/freertos.c` each need exactly one include and
one function call (`App_Init()` / `App_CreateTasks()`). See
[App/drivers/README.md](App/drivers/README.md#wiring-app_init--app_createtasks)
for the exact wiring.

The driver library itself — its architecture, the `drv_constants.h` and
`drv_platform` contracts, CMake options, interrupt wiring, the LVGL adapter,
bare-metal usage, testing, and the full CubeMX peripheral setup steps — is
documented in **[App/drivers/README.md](App/drivers/README.md)**. That
directory is a self-contained, CubeMX-free library that happens to live here;
drop it into `App/drivers/` of any other project the same way.

## Build

Presets target the STM32CubeCLT `arm-none-eabi-gcc` toolchain via
`Profiles/board_1/cmake/gcc-arm-none-eabi.cmake` (falls back to whatever
`arm-none-eabi-gcc` is on `PATH` if the CubeCLT install path isn't found).

```bash
cmake --preset Debug
cmake --build --preset Debug
```

```bash
cmake --preset Release
cmake --build --preset Release
```

Both presets pass `-DPROFILE=board_1` (the default). Flashing (from
`.vscode/tasks.json`):

```bash
STM32_Programmer_CLI -c port=SWD -w build/Debug/01_Display.elf -v -rst
```

## Porting to a new board

1. `cp -r Profiles/board_1 Profiles/board_2` — or generate fresh from CubeMX
   into `Profiles/board_2/`.
2. Copy and edit `App/drv_constants.h` for the new pin/handle mapping.
3. Build with `-DPROFILE=board_2`.

No changes to `App/drivers/` (the library) are needed — board variation
lives entirely in `drv_constants.h` content. See
[App/drivers/README.md](App/drivers/README.md#porting-to-a-new-board) for
more detail.

## Testing

Host unit tests for the driver library (no ARM toolchain needed):

```bash
cd App/drivers/tests/host && ./run.sh
```

The on-target hardware smoke checklist lives at
`App/drivers/tests/target/hardware_smoke_checklist.md`.
